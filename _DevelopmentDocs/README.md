# SGG PerfMeter Development Docs

Этот каталог предназначен только для внутренней разработки пакета. Пользовательская документация живет в `docs/en`, `docs/ru` и package-local `.Documentation`.

## Структура

| Раздел | Назначение |
| --- | --- |
| `backlog/` | Полезные, но еще не реализованные задачи и направления. |
| `decisions/` | Короткие технические решения: что поддерживаем, что не поддерживаем и почему. |
| `release/` | Минимальные внутренние материалы для подготовки release candidate и public release. |
| `agents/` | Внутренние локальные заметки для агентов; корневой `AGENTS.md` остается публичным guide. |

## Правила

- Не хранить здесь пользовательские guides или публичные продуктовые тексты.
- Не хранить историю уже завершенных итераций, если результат вошел в release notes или changelog.
- Backlog должен описывать только будущую работу и текущие ограничения.
- Release-доки должны быть короткими и актуальными для текущего release pass.

## Стиль Документации

- Не использовать эмодзи в пользовательской или development-документации, кроме согласованных status icons в comparison-таблицах: `🔵`/`✔` - активно или успех, `▲`/`⚠️` - предупреждение, `●`/`🛑` - ошибка или опасность, `◇` - опционально.
- Избегать конструкций вида `Это не ... Это ...`, `not just ...`, `not a ... but ...`; формулировать через прямое позитивное описание пользы, scope и ограничений.
