## Рекомендации

- Сначала делайте читаемые desktop-скриншоты, а примеры для мобильных устройств добавляйте после проверки на устройствах.
- По возможности используйте реальные Unity-сцены вместо пустых тестовых сцен.
- Не используйте локальные изображения из `.ScreenReferences/` напрямую; это дизайн-референсы, а не скриншоты продукта.
- Держите скриншоты достаточно компактными для GitHub и документации пакета.
